## Level 1：最小语言模型

**目标：** 理解"训练一个语言模型"到底是什么意思——用最简单的模型说明整个训练循环。

**核心资源：** Karpathy "makemore" Part 1（视频 2 小时）

**链接：** https://www.youtube.com/watch?v=PaCmpygFfXo
**配套代码：** https://github.com/karpathy/makemore

### 你要做的

**第 1 天：Bigram 计数模型**

从大约 15 行代码开始。数据集是 `names.txt`——32,000 个英文名字。

```python
# 最粗暴的 bigram 语言模型：统计每对字母的频率
import torch

words = open('names.txt').read().splitlines()
chars = sorted(list(set(''.join(words))))
stoi = {s:i+1 for i,s in enumerate(chars)}
stoi['.'] = 0

# 构建计数矩阵
N = torch.zeros((27, 27), dtype=torch.int32)
for w in words:
    chs = ['.'] + list(w) + ['.']
    for ch1, ch2 in zip(chs, chs[1:]):
        N[stoi[ch1], stoi[ch2]] += 1

# 从频率中采样
P = (N+1).float()  # +1 是平滑
P /= P.sum(1, keepdim=True)

g = torch.Generator().manual_seed(42)
for i in range(5):
    out = []
    ix = 0
    while True:
        p = P[ix]
        ix = torch.multinomial(p, 1, generator=g).item()
        out.append(itos[ix])
        if ix == 0:
            break
    print(''.join(out))
```

**你要理解的关键问题：**
- 这个模型为什么能生成看起来像英文名字的字符串？
- 损失函数是什么？（负对数似然 = 模型的"惊讶程度"）
- 为什么训练 loss 越低的模型生成质量越好？

**第 2 天：梯度版本**

把同样的 bigram 模型改写为神经网络版本——使用 Embedding + CrossEntropyLoss + 梯度下降：

```python
# 神经网络版 bigram
W = torch.randn((27, 27), requires_grad=True)

for step in range(1000):
    # forward
    logits = W[x]          # x 是 input 字符的索引
    loss = F.cross_entropy(logits, y)  # y 是下一个字符
    
    # backward
    W.grad = None
    loss.backward()
    
    # update
    W.data -= 50.0 * W.grad
```

**你要对比：计数模型和梯度模型学习到的是同样的东西吗？loss 最终收敛到相同值吗？**

### Level 1 完成标志

- [ ] 你跑通了 bigram 计数模型，生成了看起来像名字的字符串
- [ ] 你跑通了梯度版本的 bigram，loss 从 ~3.3 降到 ~2.5
- [ ] 你能解释 "forward → loss → backward → update" 每一步在做什么
- [ ] 你理解了负对数似然：`-log(P)` 越大 = 模型越"惊讶" = 预测越差

## Level 2：MLP + Embedding

**目标：** 理解"向量表示"和"非线性层"——这是从 bigram 到 Transformer 中间必经的一步。

**核心资源：** Karpathy "makemore" Part 2（视频 1 小时 15 分）

**链接：** https://www.youtube.com/watch?v=TCH_1BHY58I

### 你要做的

从 bigram（只根据上一个字符预测）升级到**根据前 3 个字符预测**。

**关键代码骨架：**

```python
C = torch.randn((27, 10))          # embedding 表：27 字符 → 10 维向量
W1 = torch.randn((30, 200))        # 输入层：3个字符×10维 = 30 → 200 隐藏神经元
b1 = torch.randn(200)
W2 = torch.randn((200, 27))        # 输出层：200 → 27（词汇表大小）
b2 = torch.randn(27)

parameters = [C, W1, b1, W2, b2]
for p in parameters:
    p.requires_grad = True

for step in range(100000):
    # forward
    emb = C[X]                      # (batch, 3, 10)
    h = torch.tanh(emb.view(-1, 30) @ W1 + b1)  # 非线性激活
    logits = h @ W2 + b2
    loss = F.cross_entropy(logits, Y)
    
    # backward + update
    for p in parameters:
        p.grad = None
    loss.backward()
    lr = 0.1 if step < 50000 else 0.01
    for p in parameters:
        p.data -= lr * p.grad
```

**你要观察的：**
- Embedding 可视化：训练后，相似字符（如 `a` 和 `o`——都是元音）的 embedding 向量在空间中是否靠近？
- tanh 的作用：去掉 tanh 会怎样？`h` 的形状会变吗？loss 会变吗？
- 上下文长度的影响：从 3 个字符变到 5 个，loss 改善多少？变到 1 个呢？

### Level 2 完成标志

- [ ] 你跑了 MLP 版本，loss 从 bigram 的 2.5 降到 ~2.2
- [ ] 你理解了 embedding 就是"查表"——字符索引 → 稠密向量
- [ ] 你理解了 tanh 为什么是"非线性"的——去掉它模型就退化回线性 bigram

## Level 3：注意力机制——这是核心

**目标：** 从零构建自注意力，建立"代码先于公式"的深层直觉。

**核心资源：**
- Karpathy "makemore" Part 5（WaveNet + 注意力）或 nanoGPT 视频的注意力部分（约 30 分钟）
- Jay Alammar "[The Illustrated Transformer](https://jalammar.github.io/illustrated-transformer/)"（可视化）

### 第 1 步：平均聚合——注意力的前身

先不要看注意力公式。从最简单的东西开始——取前面所有字符 embedding 的平均值：

```python
# 对于序列中的每个位置，取它之前所有 token 的平均 embedding
# 序列: a b c d
# 位置 2 ('c') 的输出 = (emb(a) + emb(b) + emb(c)) / 3
# 位置 3 ('d') 的输出 = (emb(a) + emb(b) + emb(c) + emb(d)) / 4

B, T, C = 4, 8, 32  # batch=4, 序列长度=8, embedding 维度=32
x = torch.randn(B, T, C)

# 用 for 循环版本（让你看清楚）
xbow = torch.zeros(B, T, C)
for b in range(B):
    for t in range(T):
        xprev = x[b, :t+1]          # 取当前及之前的所有 token
        xbow[b, t] = torch.mean(xprev, 0)  # 求平均
```

**现在理解"权重"的含义。** 在上述平均聚合中，每个之前位置被赋予相同的权重 `1/t`。但"权重相同"是一个非常弱的假设——对预测来说，某些前面位置的词显然比其他的更重要。

自注意力的核心就是：**让模型自己学习每个位置的权重**。

### 第 2 步：矩阵乘法实现加权聚合

```python
# 用 softmax 后的矩阵乘法替代 for 循环的平均
# 构造下三角矩阵作为"权重的初始猜测"
tril = torch.tril(torch.ones(T, T))      # 下三角矩阵
wei = torch.zeros(T, T)
wei = wei.masked_fill(tril == 0, float('-inf'))  # 未来位置权重为 0
wei = F.softmax(wei, dim=-1)              # 每行归一化
out = wei @ x                              # 加权聚合
```

此时 `wei` 矩阵每行是均匀的（因为初始权重全是 0，softmax 后均匀分配）。这等价于前面的平均聚合——所有位置权重相同。

### 第 3 步：引入 Query 和 Key——让权重可学习

**这才是自注意力的核心创新：**

```python
# 之前：wei = softmax(mask(zeros))，每行均匀
# 现在：wei = softmax(mask(Q @ K^T))，权重由数据决定

head_size = 16
query = nn.Linear(C, head_size, bias=False)
key   = nn.Linear(C, head_size, bias=False)
value = nn.Linear(C, head_size, bias=False)

q = query(x)   # (B, T, head_size)  —— "我在找什么"
k = key(x)     # (B, T, head_size)  —— "我能提供什么"
v = value(x)   # (B, T, head_size)  —— "我的实际内容"

wei = q @ k.transpose(-2, -1)        # (B, T, T) —— 每对位置的"匹配分数"
wei = wei / (head_size ** 0.5)       # 缩放！防止 softmax 饱和
wei = wei.masked_fill(tril == 0, float('-inf'))  # 不能看未来
wei = F.softmax(wei, dim=-1)         # 归一化为概率

out = wei @ v                         # 加权聚合 value
```

**直觉检验——在你读完这个代码后，应该能回答：**

1. 假设序列是 "the cat sat on the mat"，位置 3（"sat"）应该"注意"位置 1（"cat"）还是位置 2（"the"）更多？为什么？
2. 如果 q[3] 和 k[1] 的点积很大，意味着什么？
3. 为什么需要 `head_size ** 0.5` 这个缩放？去掉它试试——softmax 的输出会变成什么样？

### 第 4 步：Multi-Head——多个"视角"

```python
class MultiHeadAttention(nn.Module):
    def __init__(self, n_embd, n_head, block_size):
        super().__init__()
        head_size = n_embd // n_head
        self.heads = nn.ModuleList([
            Head(n_embd, head_size, block_size) for _ in range(n_head)
        ])
        self.proj = nn.Linear(n_embd, n_embd)
    
    def forward(self, x):
        # 每个 head 独立计算注意力，结果拼接
        out = torch.cat([h(x) for h in self.heads], dim=-1)
        return self.proj(out)
```

**直觉：** 不同的头学习不同的"关注模式"。一个头可能关注语法依存（形容词 → 名词），另一个关注长距离指代（"it" → 前文提到的那个东西），第三个关注位置编码。

### Level 3 完成标志

- [ ] 你手写出了 self-attention 的单头版本，loss 比纯 MLP 更低
- [ ] 你写出 multi-head 版本，看到 loss 继续下降
- [ ] 你能用一句话解释 Q、K、V 各自的职责
- [ ] 你理解 `softmax(QK^T/√d_k)V` 就是"用 Q 和 K 的匹配度作为权重，对 V 做加权平均"
- [ ] **现在打开 Jay Alammar 的 Illustrated Transformer，你应该觉得它全在说废话——因为你已经亲手写过每一步了**

## Level 4：完整 Transformer（nanoGPT）

**目标：** 组装所有组件，训练一个完整的小型 GPT。

**核心资源：** Karpathy "Let's build GPT: from scratch, in code, spelled out."（视频 2 小时）

**链接：** https://www.youtube.com/watch?v=kCc8FmEb1nY
**配套仓库：** https://github.com/karpathy/nanoGPT

### Transformer Block 的完整结构

在 Level 3 的基础上，加上剩下的组件：

```python
class Block(nn.Module):
    def __init__(self, n_embd, n_head, block_size):
        super().__init__()
        self.ln1 = nn.LayerNorm(n_embd)      # 训练稳定器
        self.attn = MultiHeadAttention(n_embd, n_head, block_size)
        self.ln2 = nn.LayerNorm(n_embd)
        self.mlp = nn.Sequential(
            nn.Linear(n_embd, 4 * n_embd),   # 放大 4 倍
            nn.GELU(),                         # 非线性
            nn.Linear(4 * n_embd, n_embd),   # 缩回原大小
        )
    
    def forward(self, x):
        # 注意：Pre-Norm —— LayerNorm 在 attention/MLP 之前
        x = x + self.attn(self.ln1(x))   # 残差连接 + 注意力
        x = x + self.mlp(self.ln2(x))    # 残差连接 + MLP
        return x
```

### 训练过程中你要观察的

- **Loss 曲线**：从 ~4.0 开始下降到 ~1.5 左右。如果训练到 1.5 以下还未饱和，说明模型容量还有提升空间。
- **生成质量的变化**：每隔 500 步采样一次，观察输出如何从随机字符逐渐变成"看起来像英语"的序列。
- **梯度范数**：如果突然飙升，可能是学习率太大或数据中有异常值。

- **Section 3.2.2（Multi-Head Attention）** ：你实现了。理解 `head_i = Attention(QW^Q_i, KW^K_i, VW^V_i)` 就是你在 `nn.ModuleList([Head() for _ in range(n_head)])` 里做的事。

### 你现在读 InstructGPT（Ouyang et al. 2022）的方式

## 一个最重要的建议

**不要提前读论文。**

论文是知识的压缩形式——它假设读者已经拥有大量的隐含背景知识。在 Level 3 完成之前打开 "Attention Is All You Need"，那些公式对你来说只是一个需要记忆的符号序列。

正确的顺序是：**用代码构建直觉 → 用论文形式化直觉 → 用代码验证论文**。

当你写出 `q @ k.transpose(-2, -1)` 并看到 loss 下降的那一刻，你不需要"记住"注意力公式——你知道它为什么长这样。
