## 4. 逐模块深度分析

### 4.1 Transformer 架构

**一手资料。** Vaswani et al.（2017），"Attention Is All You Need"，NIPS 2017。arXiv:1706.03762。

#### 4.1.1 自注意力机制：核心公式

Transformer 的核心运算是 **Scaled Dot-Product Attention**：

\[
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
\]

拆解这个公式的每一步：

1. **Q（Query）· K（Key）· V（Value）** 是从同一输入 X 通过不同的权重矩阵投影得到的：
   \[
   Q = X W_Q,\quad K = X W_K,\quad V = X W_V
   \]
   每个权重矩阵维度为 `d_model × d_k`（或 `d_model × d_v`）。

2. **QK^T** 计算每对位置之间的"相关度分数"。结果是一个 `N × N` 矩阵（N = 序列长度）。位置 (i, j) 的含义是"位置 i 的 Query 与位置 j 的 Key 有多匹配"。

3. **除以 sqrt(d_k)**。论文的关键发现：当 d_k 较大时，QK^T 的值的方差会增大，导致 softmax 进入梯度极小的饱和区。除以 sqrt(d_k) 将方差控制为 1，保证了训练稳定性。

4. **Softmax** 将分数归一化为概率分布。注意：softmax 在每一行上独立计算——每个 Query 独立地在所有 Key 上分配权重。

5. **乘以 V**。Softmax 输出的注意力权重矩阵（N × N）与 V（N × d_v）相乘：每个位置的输出 = 所有位置的 Value 的加权和，权重由注意力分数决定。

**为什么叫"自注意力"。** 因为 Q、K、V 都来自同一个序列（Self）。与此相对的是 Encoder-Decoder Attention：Q 来自 Decoder，K、V 来自 Encoder 的输出。

#### 4.1.2 Multi-Head Attention

论文使用 h = 8 个注意力头并行计算，每个头的维度为 d_k = d_v = d_model / h = 64：

\[
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W_O
\]
\[
\text{head}_i = \text{Attention}(QW_{Q_i}, KW_{K_i}, VW_{V_i})
\]

**为什么要多头？** 不同头学习不同的关系模式。一个头可能关注语法关系（如形容词-名词），另一个关注远距离的指代关系，第三个关注局部上下文。多头类似于卷积网络中的多个滤波器——不同"通道"捕获不同特征。

#### 4.1.3 Causal Mask（因果遮罩）

Decoder-only 模型（GPT）的核心约束。在训练时：

```
在预测位置 t 的词时，模型只能看到位置 1 到 t-1 的词（左边），
不能看到位置 t+1 到 N 的词（右边）。
```

实现方式：在 softmax 之前，将 QK^T 矩阵中所有 i < j 的位置（即"未来"位置）置为 -∞，使得 softmax 后这些位置的注意力权重为 0。

这与 BERT 的**双向注意力**形成了根本区别。BERT 可以看到左右两侧的所有词，因此 BERT 适合"理解"任务（分类、QA）；GPT 的因果注意力使其天然适合"生成"——从左到右预测下一个词。

#### 4.1.4 Layer Normalization 的位置

这是一个微妙的架构选择，影响训练稳定性：

| 位置 | 含义 | 使用模型 |
|------|------|----------|
| Post-Norm | 残差连接后做 LayerNorm：x + Sublayer(Norm(x)) | 原版 Transformer、GPT-1 |
| Pre-Norm | 残差连接前做 LayerNorm：Norm(x + Sublayer(x)) | GPT-2 起、Llama 系列 |

GPT-2 转向 Pre-Norm 的原因：Pre-Norm 在训练初期梯度更稳定，允许更大的学习率，且在深层网络中不容易产生梯度爆炸。这在实践中如此重要，以至于后续几乎所有大型模型都采用了 Pre-Norm。

#### 4.1.5 位置编码的演进

Transformer 本身没有"顺序"的概念——它并行处理整个序列，需要额外的机制注入位置信息。

| 方法 | 来源 | 特点 |
|------|------|------|
| **Sinusoidal** | Vaswani et al., 2017 | 固定频率的正弦/余弦函数，可以泛化到训练时未见的长度 |
| **Learned** | GPT-1（Radford et al., 2018） | 可学习的位置嵌入矩阵，长度上限由训练决定 |
| **RoPE（Rotary Position Embedding）** | 由 Llama 系列推广 | 通过旋转矩阵将位置信息编码到 Q、K 中，具有相对位置编码的性质。Llama 2 采用 |

RoPE 的数学直觉：将 token 的 embedding 在二维平面上旋转一个角度，旋转角度正比于 token 在序列中的位置。两个 token 的内积自然携带了它们的相对位置信息。

#### 4.1.6 FlashAttention

**一手资料。** Dao et al.（2022），"FlashAttention: Fast and Memory-Efficient Exact Attention with IO-Awareness"，NeurIPS 2022。arXiv:2205.14135。

**问题。** 标准 attention 的 N × N 注意力矩阵占据 O(N²) 显存。对于 N=4096，矩阵为 4096 × 4096 ≈ 16M 个浮点数（FP16 下约 32MB）；对于长上下文（128K），矩阵膨胀到约 16G 个元素，单个 GPU 无法承受。

**关键洞见。** FlashAttention 不是修改注意力计算的数学结果（它计算的结果**完全等价**于标准 attention），而是重新组织计算的 GPU 内存布局。核心思想是 **IO 感知**——标准 attention 的瓶颈不是 FLOPs（计算量），而是 HBM（高带宽内存）的读写次数。

**具体做法。**
1. **Tiling（分块）**：将 Q、K、V 分割为小块，在芯片上的 SRAM（比 HBM 快 10 倍以上）中逐块计算 softmax，N × N 矩阵不写入 HBM。
2. **Recomputation（重计算）**：反向传播时不保存中间矩阵，而是利用前向传播保存的统计量（softmax 的 m 和 ℓ 值）在反向传播时重新计算。

**效果。**
- HBM 访问量从 O(N²d) 降至 O(N²d²/M)。
- 训练 BERT-large 比 MLPerf 记录快 15%。
- GPT-2 训练比 HuggingFace 实现快 3 倍。
- 序列长度 4K 时，显存节省 20 倍。

**FlashAttention-2（Dao, 2023，ICLR 2024）。** 进一步将 GPU 利用率从 25–40% 提升到 50–73%。成为 PyTorch 2.0 标准 attention 后端，并被 Llama、Mistral 等所有主流模型采用。

### 4.2 Tokenization：文本如何变成数字

**一手资料。** Sennrich et al.（2016），"Neural Machine Translation of Rare Words with Subword Units"（BPE）；Kudo & Richardson（2018），"SentencePiece: A simple and language independent subword tokenizer and detokenizer for Neural Text Processing"。

#### 4.2.1 为什么需要子词分词

LLM 不能直接处理原始文本。Tokenizer 将文本转换为模型可以消费的整数 ID 序列。挑战在于：

- **词汇量限制**：词表不能无限大。英文单词的形态变化（run/runs/running/ran）、合成词、专业术语、以及多语言场景使基于完整单词的词汇表不可能覆盖所有情况。
- **OOV（Out-of-Vocabulary）问题**：如果词表只包含最常见的 50,000 个单词，长尾词和稀有词会遇到"未知词"标记，丢失信息。

子词分词的解决方案：将稀有词拆分为常见子词，如 `"unbelievable"` 拆为 `["un", "believ", "able"]`。所有子词都在词表中，因此永远不会遇到未知词。

#### 4.2.2 各模型使用的分词方法

| 方法 | 代表模型 | 词表大小 | 特点 |
|------|----------|----------|------|
| **BPE（Byte-Pair Encoding）** | GPT-1 | 40,000 | 基于字符对的合并频率 |
| **Byte-level BPE** | GPT-2/3 | 50,257 | 在字节层面操作，可处理任意文本 |
| **WordPiece** | BERT | 30,522 | 基于 likelihood 而非频率的合并策略 |
| **SentencePiece（BPE）** | Llama 2 | 32,000 | 将空格编码为特殊字符，完全可逆 |

**BPE 算法直觉。**
1. 将每段文本拆分为字符序列。
2. 统计所有相邻字符对的出现频率。
3. 将最高频的字符对合并为一个新的子词 token。
4. 重复步骤 2–3，直到词汇表达到目标大小。

Byte-level BPE 的突破在于：它直接在 UTF-8 字节上操作，而非在 Unicode 字符上。这意味着词表天然覆盖了所有语言——任何文本（中文、emoji、罕见字符）由一个 256 个基础 byte token 的序列表示。GPT-2 从此开始不再有"未知词"问题。

#### 4.2.3 分词对模型能力的实际影响

- **数学/编码能力**：Llama 2 将数字分割为单个数字，使模型能够"计算"数字位数。GPT-2 的 byte-level BPE 将数字作为完整 token 处理，导致"137+256"这样的任务需要理解数字的内部结构。
- **多语言能力**：词表大小固定时，分配给某语言的 token 越多，该语言的效率越高。这就是为什么在非英语语言上 token 效率较低——同样的意思需要更多 token 来表示。
- **Token 计数与成本**：API 计费按 token 计算。中文字符通常 1 个字符 = 1-2 个 token，英文约为 0.75 个词 = 1 个 token。

### 4.3 训练范式：预训练 → 微调 → 对齐

现代 LLM 的训练通常经历三个阶段。每个阶段消耗的计算量和数据量跨越数个数量级。

#### 4.3.1 预训练

**一手资料。** GPT-1（Radford et al., 2018）；GPT-2（Radford et al., 2019）；GPT-3（Brown et al., 2020）；Llama 2（Touvron et al., 2023）。

预训练是整个过程的主体——消耗 99% 以上的计算资源。目标是一个：**下一词预测**。

**以 GPT-3 为例（Brown et al., 2020）：**

| 超参数 | 值 |
|--------|-----|
| 参数量 | 175B |
| 层数 | 96 |
| 隐层维度 | 12,288 |
| 注意力头 | 96 |
| 上下文长度 | 2,048 |
| 训练数据 | 约 300B token（CommonCrawl 60% + WebText2 22% + 书籍 16% + Wikipedia 3%） |
| Tokenizer | Byte-level BPE，词表 50,257 |
| 优化器 | Adam |
| 批量大小 | 约 3.2M token |

**GPT-3 的训练数据混合是经过精心设计的。** CommonCrawl 是最丰富但噪声最大的数据源。原始 CommonCrawl 首先经过分类器过滤（训练了一个模型来区分 CommonCrawl 文档和 WebText 文档的质量），然后进行模糊去重。WebText2 质量较高但数量有限。书籍提供连贯的长文本和丰富知识。Wikipedia 提供高质量的事实验证。数据集的最终混合比例并非随意选择——OpenAI 通过实验发现这个配方效果最好。

**以 Llama 2 为例（Touvron et al., 2023）：**

| 超参数 | 值 |
|--------|-----|
| 最大参数量 | 70B |
| 训练数据 | 2T token（远超 GPT-3 的 300B） |
| Context 长度 | 4,096（比 GPT-3 的 2,048 翻倍） |
| 优化器 | AdamW（β₁=0.9, β₂=0.95） |
| 峰值学习率 | 3.0×10⁻⁴（7B/13B/70B） |
| LR Schedule | Cosine，2,000 步 warmup |
| 梯度裁剪 | 1.0 |
| 批量大小 | 4M token |
| GPU | NVIDIA A100-80GB |
| 总 GPU 时间 | 约 3.3M GPU-hours |

**关键观察：** Llama 2 训练了 2T token 后，loss 曲线**仍然没有饱和**。这与 Chinchilla 缩放定律一致（见 §4.5）。

#### 4.3.2 监督微调（SFT）

**一手资料。** Ouyang et al.（2022），InstructGPT。

预训练给了模型"语言能力"——流畅的生成、丰富的知识。但预训练模型的输出方向是不可控的。给它一个 prompt，它可能继续生成、总结、翻译或编造——它没有"意图"的概念。

SFT 改变的是输出行为。微调数据的形式是：
```
{"prompt": "解释什么是黑洞", "response": "黑洞是..."}
{"prompt": "写一封感谢邮件", "response": "尊敬的..."}
```

**关键洞察来自 InstructGPT 和 Llama 2 的实践：数据质量远胜于数量。**

- InstructGPT 使用了约 13,000 条人工标注的高质量 SFT 数据。
- Llama 2 使用了 27,540 条 SFT 数据。
- 两者都发现，几万条精心编写的样本就足够，几十万条第三方自动生成的样本反而可能降低质量。

SFT 的"魔法"在于：模型在预训练阶段已经知道"如何写一封好邮件"，只是不知道"当用户要求写邮件时应该写邮件"。SFT 不是教模型新知识——而是对齐模型的"意图"和用户的"期望"。

### 4.4 下一词预测：语言建模的数学本质

虽然下一词预测（Next Token Prediction）看似简单——"给定前面的文本，猜下一个词是什么"——但其隐含的数学结构和能力来源值得深入理解。

#### 4.4.1 自回归框架

令文本序列为 X = (x₁, x₂, ..., x_N)，其中每个 xᵢ 是词表中的一个 token ID。模型的目标是学习条件概率分布：

\[
P(x_1, x_2, ..., x_N) = \prod_{t=1}^{N} P(x_t \mid x_1, x_2, ..., x_{t-1})
\]

模型在训练时接收整个序列，但每个位置的输出只应该预测**该位置的下一个** token。因此，交叉熵损失函数为：

\[
\mathcal{L} = -\frac{1}{N}\sum_{t=1}^{N} \log P(x_t \mid x_1, ..., x_{t-1}; \theta)
\]

#### 4.4.2 为什么"猜下一个词"足以产生所有能力

从信息论的角度看，下一词预测是一个极其强大的学习信号。为了准确预测下一个词，模型必须隐式学会：

- **语法**：预测连续两个词是否合法需要句子结构的知识。
- **事实**：预测"The capital of France is __"要求模型知道巴黎。
- **推理**：预测一个数学推导的结论需要逐步逻辑。
- **常识**：预测一件日常事件的下一步需要理解世界的因果规律。

GPT-3 论文的标题本质上就是在陈述这一发现："语言模型是少样本学习者"——一个设计目标仅为预测下一个词的模型，在规模足够大后涌现出了一个完全未预料到的能力：**meta-learning**。给定输入中的几个示例，模型可以推断出模式并应用到新实例上，而无需任何参数更新。

#### 4.4.3 与 Masked Language Model 的对比

BERT（Devlin et al., 2018）使用不同的训练目标：MLM（Masked Language Modeling）。随机遮罩 15% 的 token，模型预测被遮罩的 token。

```
输入：  The [MASK] is shining today.
目标：  预测 [MASK] = "sun"
```

**关键区别：**

| | GPT（自回归） | BERT（MLM） |
|---|---|---|
| **注意力方向** | 只能看左边（Causal） | 可以看左右两边（Bidirectional） |
| **预测方式** | 一次一个 token，顺序生成 | 每个位置独立预测 |
| **天然适合** | 文本生成 | 文本理解（分类、QA） |
| **训练效率** | 每个 token 产生一个训练信号 | 仅 15% 的 token 产生训练信号 |
| **代表模型** | GPT-1/2/3/4, Llama 系列 | BERT, RoBERTa, DeBERTa |

为什么 GPT 的范式最终主导？因为**生成能力**使 GPT 可以处理所有 BERT 能处理的任务（将分类转化为"生成答案"），而反过来不行。

### 4.5 缩放定律：模型、数据与计算预算

#### 4.5.1 Kaplan 等人（2020）— OpenAI 缩放定律

**一手资料。** Kaplan et al.（2020），"Scaling Laws for Neural Language Models"。arXiv:2001.08361。

**核心发现：** 交叉熵损失（衡量下一词预测的难度）与模型参数量 N、数据量 D、计算量 C 呈现跨越 7 个数量级的**幂律关系**：

\[
L(N, D, C) \propto N^{-\alpha_N},\ D^{-\alpha_D},\ C^{-\alpha_C}
\]

这意味着，无论模型现在多大，只要增加更多参数或数据，性能就会以可预测的方式继续提升。这是"大力出奇迹"的数学基础。

**论文的具体结论：**

| 发现 | 内容 |
|------|------|
| **幂律缩放** | Loss = (N)^{-\alpha_N}，α_N ≈ 0.076。参数量每翻 10 倍，loss 按一定比例下降 |
| **架构独立性** | 当固定非嵌入参数量 N 时，性能对深度、宽度、注意力头数等超参数不敏感 |
| **计算最优分配** | N ∝ C^{0.73}, D ∝ C^{0.27}。这意味着：**把更多预算花在大模型上，而非在更多数据上训练** |
| **样本效率** | 更大的模型是更好的学习者——达到相同 loss 需要的步骤更少 |
| **收敛前停止** | 计算最优策略=训练非常大的模型，在远未收敛时就停下来 |

**后果。** 在 2020–2022 年间，这个结论引发了"更大更好"的军备竞赛：GPT-3 = 175B 参数，训练了 300B token。按照 Kaplan 法则，这是一个合理的选择——把预算倾斜给模型大小而不是数据量。

#### 4.5.2 Hoffmann 等人（2022）— Chinchilla 修正

**一手资料。** Hoffmann et al.（2022），"Training Compute-Optimal Large Language Models"。NeurIPS 2022。arXiv:2203.15556。

DeepMind 用三种不同的方法重新估计了缩放定律，得出了与 Kaplan 截然不同的结论：

**核心修正：**

| | Kaplan et al.（2020） | Hoffmann et al.（2022） |
|---|---|---|
| **最优 N ∝ C^a** | C^{0.73}（模型优先） | C^{0.50}（同等重要） |
| **最优 D ∝ C^b** | C^{0.27}（数据次要） | C^{0.50}（同等重要） |
| **最优 Token/Param 比** | ≈ 1.7 | ≈ **20** |
| **训练策略** | 做大模型但不充分训练 | 模型和数据等比例扩大 |

**实证验证。** DeepMind 训练了 Chinchilla——一个 70B 参数的模型，在 1.4T token 上训练。与 GPT-3（175B 参数，300B token）使用相同的计算预算（注意：175B × 300B ≈ 53e21 FLOPs，而 70B × 1.4T ≈ 98e21，总体计算开销接近）。Chinchilla 在所有下游基准上全面超越 GPT-3，同时体积更小——更容易部署。

**影响。** 这个修正从根本上改变了 LLM 训练的范式。Llama 2（2T token）、Mistral 等后续开源模型都遵循了"数据与模型同等重要"的 Chinchilla 原则。

#### 4.5.3 缩放定律的实际含义

```
Kaplan 公式：做 175B 参数的模型，训练 300B token。
Chinchilla 公式：同样的计算预算，应该做 70B 模型，训练 1.4T token。
后者在各方面的表现都更好。
```

对于学习者的关键启示：**模型大小不是唯一的变量。数据质量和数据量同样重要，甚至更重要。**

### 4.6 RLHF 与 DPO：如何让模型"听话"

#### 4.6.1 InstructGPT / RLHF 三阶段管线

**一手资料。** Ouyang et al.（2022），"Training Language Models to Follow Instructions with Human Feedback"。NeurIPS 2022。arXiv:2203.02155。

InstructGPT 是 ChatGPT 的前身。其方法被称为 RLHF —— Reinforcement Learning from Human Feedback。

**三阶段管线：**

```
Stage 1: SFT（监督微调）
  人工标注员写下"理想回答"。
  训练模型在这些数据上模仿人工回答。

Stage 2: RM（Reward Model）训练
  模型对同一个 prompt 生成多个回答。
  人工标注员对回答进行质量排序（A > B > C > D）。
  训练一个奖励模型来预测人工偏好：给定 (prompt, response)，输出标量分数。

Stage 3: PPO（Proximal Policy Optimization）
  用奖励模型给模型生成的回答打分。
  使用强化学习（PPO 算法）优化模型参数以最大化这个分数。
  同时添加 KL 惩罚，防止模型偏离 SFT 阶段的"合理分布"太远。
```

**PPO 的损失函数：**

\[
R(g \mid p) = \tilde{R}_c(g \mid p) - \beta \cdot D_{KL}(\pi_\theta \parallel \pi_{\text{SFT}})
\]

- 第一项是 Reward Model 给的分数。
- 第二项是 KL 散度：衡量当前模型 π_θ 的输出分布与 SFT 模型 π_SFT 的输出分布之间的差异。
- β 控制折中程度：太大则模型不会改进；太小则模型可能输出高奖励但无意义的文本。

**关键结果（惊人）：** InstructGPT 的 **1.3B** 参数模型在人工偏好评估中击败了 **175B** 的 GPT-3。对齐的价值可以弥合 100 倍的参数量差距。

#### 4.6.2 DPO：直接偏好优化

**一手资料。** Rafailov et al.（2023），"Direct Preference Optimization: Your Language Model is Secretly a Reward Model"。NeurIPS 2023 Oral。arXiv:2305.18290。

**问题。** RLHF 管线复杂且不稳定：
- 需要单独训练一个 Reward Model。
- PPO 有多个超参数（KL 系数、价值函数学习率、GAE lambda 等）——调参成本极高。
- 训练不稳定：奖励模型不完美，PPO 倾向于利用奖励模型的"作弊"弱点。

**DPO 的关键洞见。** 将 RLHF 的目标从强化学习重新表述为一个**直接的分类问题**。推导过程：

1. 从 RLHF 的目标函数出发（最大化奖励，约束 KL 散度）。
2. 推导出最优策略的闭式解，发现：
   \[
   \pi^*(y \mid x) = \frac{1}{Z(x)} \pi_{\text{ref}}(y \mid x) \exp\left(\frac{1}{\beta} r(x, y)\right)
   \]
3. 将其代入 Bradley-Terry 偏好模型，得到可以直接在偏好数据上优化的损失函数：
   \[
   \mathcal{L}_{\text{DPO}} = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma\left( \beta \log\frac{\pi_\theta(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} - \beta \log\frac{\pi_\theta(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)} \right) \right]
   \]

这里 y_w 是被偏好的回答，y_l 是不被偏好的回答。损失函数的直觉是：增加偏好回答相对于参考模型的概率，减少不喜欢回答相对于参考模型的概率。

**DPO 的优势：**
- 不需要显式训练 Reward Model——语言模型**自身**就是隐含的奖励函数。
- 不需要 PPO——只需在偏好数据对上做标准分类训练。
- 训练可在一次前向传播中完成——不需要采样、不需要 rollouts。
- 被 Meta（Llama）、Mistral 等广泛采用。

**DPO 的局限：** 偏好数据仍然是必需的——模型不能自我生成偏好标签。RLHF 和 DPO 都需要人工标注偏好。DPO 简化的是优化过程而非数据需求。

### 4.7 优化与系统工程

#### 4.7.1 AdamW 优化器

**一手资料。** Loshchilov & Hutter（2019），"Decoupled Weight Decay Regularization"。ICLR 2019。

Adam 的变体。核心改进是将**权重衰减**从动量更新中解耦出来：

```
标准 Adam（L2 正则化）：
  θ_t = θ_{t-1} - η * (m_t / (√v_t + ε) + λ * θ_{t-1})

AdamW（解耦权重衰减）：
  θ_t = (1 - ηλ) * θ_{t-1} - η * m_t / (√v_t + ε)
```

在标准 Adam 中，权重衰减和自适应学习率共享归一化路径，导致正则化效果被学习率缩放。AdamW 将权重的指数衰减作为独立步骤，对每个参数施以一致的惩罚，使超参数搜索更容易。

Llama 2 的 AdamW 设置：β₁=0.9, β₂=0.95, ε=10⁻⁵, weight decay=0.1。

#### 4.7.2 学习率调度

**Cosine Schedule + Warmup** 已成为标准做法。

- **Warmup**：学习率在最初 N 步（Llama 2 为 2,000 步）从 0 线性增加到峰值。为什么需要？在训练初期，参数的随机初始化使梯度方向不稳定，warmup 避免了过大的参数更新。
- **Cosine Decay**：学习率以余弦曲线从峰值衰减到峰值的 10%。最后阶段的低学习率允许模型精细地收敛到局部最小值。

#### 4.7.3 混合精度训练

现代 LLM 训练使用 FP16（半精度浮点数）或 BF16（Google Brain Float 16）而非 FP32。

| 格式 | 指数位 | 尾数位 | 动态范围 | 精度 |
|------|--------|--------|----------|------|
| FP32 | 8 | 23 | 10^{±38} | 最高 |
| FP16 | 5 | 10 | 10^{±4} | 较低，容易溢出 |
| BF16 | 8 | 7 | 10^{±38}（与 FP32 相同） | 中等，但不会溢出 |

**为什么可以这样做。** 前向传播中，FP16/BF16 的精度足够表示大部分计算；反向传播中，使用 **loss scaling** 将梯度放大以保持在 FP16 的可表示范围内，然后在参数更新前缩小回去。BF16 的一大优势是它的动态范围与 FP32 相同，使 loss scaling 不再必要——这是它被 Llama、GPT-4 等广泛采用的原因。

#### 4.7.4 分布式训练范式

当单一 GPU 无法容纳模型时：

| 并行方式 | 原理 | 模型切片方式 |
|----------|------|-------------|
| **数据并行** | 每个 GPU 持有完整模型副本，处理不同的 batch 切片。梯度在 GPU 间同步。 | 模型不切片 |
| **模型并行** | 模型的不同层分布在不同 GPU 上。通信集中在层间边界。 | 按深度切片 |
| **流水线并行** | 在模型并行的基础上，流水线化执行以减少 GPU 空闲等待时间。 | 按深度 + 微批次 |
| **张量并行** | 矩阵乘法本身的并行化——将大矩阵运算分布到多个 GPU。 | 按矩阵维度切片 |

实际训练中（如 GPT-3 的 175B 模型），通常组合使用上述所有方式。

### 4.8 评估方法论

LLM 的评估是一个活跃的研究领域。以下是主要论文中使用的关键指标：

#### 4.8.1 Perplexity

预训练阶段的基本指标。给定一个测试集，模型对每个 token 的概率越低（越"惊讶"），loss 越高。Perplexity 指数化了 loss：

\[
\text{PPL} = \exp\left(-\frac{1}{N}\sum_{t=1}^{N} \log P(x_t \mid x_{<t})\right)
\]

直觉：如果模型对所有位置的预测都是均匀分布（极度不确定），PPL = 词汇表大小。一个完美模型 PPL = 1。实际 LLaMA 70B 的 PPL 在 WikiText 上约为 3.5–4。

#### 4.8.2 知识与推理基准

| 基准 | 衡量什么 | 代表性模型分数 |
|------|----------|---------------|
| **MMLU** | 57 个学科的大学水平知识 | Chinchilla: 67.5%, Llama 2 70B: 68.9% |
| **HellaSwag** | 常识推理（挑选最合理的句子延续） | Llama 2 70B: 87.3% |
| **HumanEval** | 代码生成能力 | Llama 2 70B: 29.3% pass@1 |
| **GSM8K** | 数学推理（小学应用题） | GPT-3 175B: 19% (5-shot) |
| **ARC-C** | 科学推理 | Llama 2 70B: 67.3% |

#### 4.8.3 人工偏好评估

在对齐阶段（RLHF/DPO），自动化基准不再充分——模型需要被**人**评估。标准方法是 **ELO 分数**（类似国际象棋评级）或**成对比较获胜率**。

InstructGPT 的关键实验：将 1.3B InstructGPT 与 175B GPT-3 进行盲测比较。人工标注员看不到哪个模型生成哪个回答。结果？**1.3B InstructGPT 被 85% 的标注员偏好于 175B GPT-3。** 证明了对齐的价值比简单的参数扩大更巨大。
